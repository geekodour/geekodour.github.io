Function pointers, dead simple example

// returns pointer to an int (returning a pointer to an int)
int *f();
// f is now a pointer to a function returning int (returning an int, but f itself is a pointer this time)
int *(f)();

arrow pointer to structures

the . operator has higher percidence than *. let's say pp is an structure pointer of some structure you declared. then,

struct mystructure *pp;

pp.x will mean (pp.x) i.e "." will get executed first. This will make no sense because x is not a pointer. to access x of pp we need to do, (pp).x, this way we first evaluate pp which points to our stucture instance then we get x from it.

This is so common that we have a shorthand, we can write (*pp).x as pp->x
